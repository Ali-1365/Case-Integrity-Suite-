## 2024-03-05 - [Preventing Unnecessary Re-renders from Polling]
**Learning:** In this codebase, polling services like `usageMonitorService.getUsage()` return brand new object references every time they are called, even if the underlying data hasn't changed. When this polling occurs inside a top-level component's `setInterval` (like `DocumentManager`), calling `setState(newData)` triggers an immediate and expensive re-render of the entire component tree every 5 seconds, simply because the object reference changed, bypassing React's `Object.is` equality check.
**Action:** When setting state from a polling service that returns objects or arrays, always use a functional state update (`setState(prev => ...)`) and implement a shallow comparison of the critical properties. If the properties haven't changed, return the `prev` reference to allow React to bailout of the render cycle.
