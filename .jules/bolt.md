## 2025-05-18 - [Backend/Network] Batch Fetch API for Praxis JSON Cache
**Learning:** Sequential, blocking synchronous `fs.readFileSync` calls on an Express event loop inside an N+1 API loop architecture significantly degrades throughput. Specifically, `lib/PraxisService.ts` fetched `/api/praxis/:lawRef` sequentially in a `for` loop, which triggered the Express server to repetitively open and parse the `praxis.json` file synchronously (`fs.readFileSync`) for each law reference. This pattern blocks concurrent request handling.
**Action:** When working with medium-sized static data files in an Express environment, always parse and store the result in an in-memory cache on the first read. Secondly, to resolve frontend network cascading (N+1), implement batched query endpoints (e.g., `/api/praxis/batch`) utilizing `express.json()` and `POST` method to bypass URL string encoding limits while minimizing the number of HTTP fetches needed.
