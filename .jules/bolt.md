## 2024-04-02 - Extracting State from Large Layouts\n**Learning:** In a heavily componentized application like this (e.g., `DocumentManager`), placing a `setInterval` or frequent state updates in top-level components causes enormous rendering cascades across the entire layout and underlying context trees.\n**Action:** Use state colocation or isolate frequently updating state into small, leaf-node components (`QuotaDisplay`) rather than putting it in the parent container. Also ensure you use proper bailout logic to prevent React from re-rendering if the relevant derived states have not actually changed.
