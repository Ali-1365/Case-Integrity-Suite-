## 2026-04-04 - [Prevent Unnecessary Re-renders from Polling]\n**Learning:** High-frequency polling (like 5-second intervals for logs or quota usage) can cause massive application-wide re-renders if the state setter is given a new object/array reference every tick, even when the underlying data is unchanged.\n**Action:** Always implement bailout logic inside `setState` (e.g., checking array length and most recent item ID, or deep equality for objects) to return the previous state reference when updating polled data.
