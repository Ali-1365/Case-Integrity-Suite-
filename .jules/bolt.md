## 2024-04-03 - [Preventing React Re-Renders from Polling]
**Learning:** The application uses `setInterval` extensively to poll for data updates (e.g., `quotaUsage`, `logs`) in top-level components or custom hooks. When simple state setters are used with objects or arrays created on the fly (like `.slice()` or constructing new `QuotaUsage` objects), it breaks React's reference equality and causes massive application-wide re-renders every 5 seconds, even if the underlying data hasn't changed.
**Action:** When implementing polling mechanisms inside `setInterval` or custom hooks, always implement bailout logic in the state setter. Use functional updates to compare the newly fetched data with the previous state (checking properties like length and most recent item IDs) and return the previous state reference if there are no functional changes.
